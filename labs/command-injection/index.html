What is OS Command Injection?

OS Command Injection is a critical vulnerability that allows attackers to execute arbitrary operating system commands on the server. This happens when an application passes unsafe user input to a system shell without proper validation. Attackers can chain malicious commands using operators like semicolons or pipes to take full control of the underlying system.

-- By Filra

How It Works?

When a user provides normal input like 8.8.8.8 to a ping function, the server executes: ping 8.8.8.8

But if the application fails to validate the input, an attacker can inject: 8.8.8.8; whoami

The server now executes both commands, revealing sensitive system information. A single unescaped parameter is all an attacker needs to transform a simple request into full remote code execution.

-- By Filra

Types of Command Injection

Direct Injection: The attacker sees the output of injected commands directly in the application response, making exploitation straightforward and easy to verify.

Blind Injection: No output is visible, but attackers confirm execution through time delays using sleep commands, file creation on accessible directories, or out-of-band DNS requests to their controlled server.

-- By Filra

Injection Operators

; Execute multiple commands sequentially regardless of whether the previous command succeeds or fails.

&& Execute the second command only if the first one succeeds without any errors.

|| Execute the second command only if the first one fails, useful for blind injection detection.

| Pipe the output of the first command as input to the second command for chaining.

-- By Filra

Impact & Risks

The impact of successful OS command injection is devastating. Attackers can steal sensitive data, delete critical files, modify system configurations, or gain complete server control through reverse shells.

Compromised servers can be used for lateral movement, pivoting to internal networks, installing cryptominers, launching DDoS attacks, or targeting other victims while hiding behind the compromised infrastructure.

-- By Filra

Real-World Cases

Log4Shell (CVE-2021-44228): Attackers injected malicious JNDI lookup commands through log messages, forcing vulnerable servers to download and execute remote code.

Shellshock (CVE-2014-6271): A critical Bash vulnerability allowed attackers to execute arbitrary commands through specially crafted environment variables on millions of web servers worldwide.

-- By Filra

How to Detect?

Time-Based: Inject sleep or ping commands with delays. If the server response is delayed by the expected time, blind command injection is confirmed.

Output-Based: Inject commands like whoami, id, or hostname and observe if their output appears in the application response directly.

Out-of-Band: Force the target server to make DNS lookups or HTTP requests to an attacker-controlled domain to exfiltrate data when direct output is not visible.

-- By Filra

Prevention Guide

Avoid system calls entirely when possible. Use built-in library functions or APIs instead of passing user input to the operating system shell.

Validate with whitelists. Only allow known safe characters like alphanumerics and dots. Reject any input containing shell metacharacters.

Run with least privilege. Execute applications under restricted accounts with minimal permissions to limit damage even if injection succeeds.

-- By Filra

Testing Tools

Burp Suite: The industry standard for manual web application penetration testing with powerful intruder and repeater tools for injection testing.

Commix: An automated open-source tool specifically designed to detect and exploit command injection vulnerabilities with various techniques.

OWASP ZAP: A free security scanner that includes automated command injection detection as part of its active scanning capabilities.
