# Contributing to Node.js-Vuln-Server

Thankful to you all for contributing

## Branch Rules

- Never commit directly to `main`.
- Create a new branch for start example banch name "feature/your-working-lab-name".
- Open a Pull Request when your work is complete.
- ME or My maintainer will review and then approve the PR before it can be merged.

## Branch Naming
### Example :
feature/upload-panel
feature/api-security
feature/react2shell
fix/upload-validation
docs/readme
etc.. 

## Commit Style

We follow Conventional Commits.

Examples:

feat(upload): add upload panel

feat(api): add upload API

fix(upload): validate MIME type

docs: update README

## Pull Requests

- Keep pull requests focused on a single feature or fix.
- Test your code before opening a PR.
- Update documentation if your changes affect users.
- Respond to review comments promptly.

## Code of Conduct
Be respectful and constructive when reviewing code.
